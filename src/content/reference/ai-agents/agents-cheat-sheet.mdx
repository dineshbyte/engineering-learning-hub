  <div class="one"><strong>The one sentence:</strong> an AI agent is a <em>stateless text-prediction
  function, wrapped in a loop, given tools and a goal, run by ordinary code.</em></div>

  <div class="cols">
    <div>
      <h2>The two diagrams</h2>
      <pre class="dgm">REQUEST PATH            THE LOOP (the engine)
User                    ┌───────────────────┐
  ↓                     ▼                   │
Agent  (LLM+loop)    OBSERVE ─▶ THINK ─▶ ACT─┘
  ↓                  (result) (LLM picks)(run tool)
Tool   (model asks,        │
  ↓     runtime runs)      ▼  goal met? → DONE
Billing Service         else loop (to max-steps)
  ↓
Database</pre>

      <h2>Definitions (compressed)</h2>
      <table>
        <tr><td>LLM</td><td>Stateless text→text fn. No memory, no hands.</td></tr>
        <tr><td>Tool</td><td>Typed fn the model may <em>request</em> (name+schema).</td></tr>
        <tr><td>Tool calling</td><td>Model emits request; <em>runtime</em> runs it.</td></tr>
        <tr><td>Agent</td><td>LLM + tools + loop + goal; model picks path.</td></tr>
        <tr><td>Agent loop</td><td>Observe→Think→Act→Repeat. Reconcile intent.</td></tr>
        <tr><td>Runtime</td><td>App server hosting loop; state, tools, limits.</td></tr>
        <tr><td>Memory</td><td>What runtime reloads. RAM=context, disk=store.</td></tr>
        <tr><td>Workflow</td><td>Dev hardcodes path. Predictable, cheap.</td></tr>
        <tr><td>Agent <span class="tag">vs WF</span></td><td>Model decides path. Flexible, costly.</td></tr>
        <tr><td>MCP</td><td>Open standard to plug tools in. USB-C for AI.</td></tr>
        <tr><td>MCP server</td><td>Exposes tools/data (the device).</td></tr>
        <tr><td>MCP client</td><td>Connects from host app (the port).</td></tr>
        <tr><td>Tool registry</td><td>Catalog advertised to model (svc discovery).</td></tr>
        <tr><td>Single agent</td><td>One loop, one context. Monolith.</td></tr>
        <tr><td>Multi-agent</td><td>Orchestrator + workers. Microservices.</td></tr>
      </table>

      <h2>Mental models</h2>
      <pre class="dgm">LLM      = Brain (no memory, no hands)
Tool     = Hands / a callable function
Agent    = Brain + Tools + Loop + Goal
Runtime  = App server that runs the loop + rules
Memory   = What runtime reloads (RAM vs disk)
Workflow = Developer chooses the path
Agent    = Model chooses the path
MCP      = USB-C / JDBC for tools
           (server=device, client=port)
Single   = Monolith    Multi = Microservices
Archetype= Same engine + diff tools + diff job</pre>

      <h2>Three rules for the VP chair</h2>
      <ol class="rules">
        <li>Default to <strong>workflows</strong>; pick agents only when the path can't be predetermined.</li>
        <li>Start <strong>single-agent</strong>; split to multi-agent like microservices — reluctantly, with evidence.</li>
        <li>To judge a pitch, ignore the brand: <strong>what tools? what loop? what guardrails?</strong></li>
      </ol>

  <h2>Ten memory rules (recite these)</h2>
  <ol class="rules">
    <li><strong>LLM</strong> — stateless text-prediction function; brain in a jar, no memory, no hands.</li>
    <li><strong>Tool</strong> — a typed function the model requests but never runs; calling = model asks, runtime does.</li>
    <li><strong>Agent</strong> — LLM + tools + loop + goal; the model picks the path.</li>
    <li><strong>Loop</strong> — a K8s reconciliation loop for intent: observe, think, act, repeat.</li>
    <li><strong>Runtime</strong> — the app server hosting the loop; holds state, runs tools, enforces guardrails.</li>
    <li><strong>Memory</strong> — what the runtime reloads into the prompt; context is RAM, the store is disk.</li>
    <li><strong>Workflow vs agent</strong> — you write the if/else vs the model writes it; choose by predictability.</li>
    <li><strong>MCP</strong> — USB-C for tools; server = device, client = port, registry = catalog.</li>
    <li><strong>Single vs multi</strong> — monolith vs microservices; split only when one context/role can't cope.</li>
    <li><strong>Archetypes</strong> — same engine + different tools + different job description.</li>
  </ol>

  <h2>5-minute review (weekly) &amp; 30-minute deep dive (when stalled)</h2>
  <p style="font-size:.84rem"><strong>5-min:</strong> recall, don't re-read — say the one sentence; draw both
  diagrams from memory; recite the ten rules; translate three buzzwords blind. <strong>30-min:</strong>
  blank-page reconstruct all ten levels; then re-derive the chain of necessity (each layer forces the
  next: stateless LLM → tools → loop → runtime → memory → workflow/agent → MCP → single/multi → archetypes);
  then place one real product on the spectrum (workflow/agent? single/multi? which archetype? which tools?).</p>
