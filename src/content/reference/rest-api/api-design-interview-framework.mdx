  <h2>The 11-step framework</h2>
  <p>Walk it in order. Each step: <strong>say or produce this</strong>, then move on. Don't boil the ocean — name the step, make the one decision that matters, signal you know the rest is there.</p>
  <ol>
    <li><strong>Clarify requirements &amp; constraints.</strong> Before any URI: who are the clients, what scale, read/write ratio, consistency needs, auth model, SLAs? <em>"Let me pin down constraints before I draw anything."</em> <a href="../lessons/what-makes-an-api-restful.html">L1</a></li>
    <li><strong>Resource model &amp; URIs.</strong> Nouns not verbs, plural collections (<code>/orders</code>), shallow nesting, opaque ids. <em>"Resources are <code>Order</code>, <code>LineItem</code>; ids are opaque."</em> <a href="../lessons/resource-modeling-and-uri-design.html">L4</a></li>
    <li><strong>Endpoints: methods + status codes.</strong> Right verb, right 2xx/4xx/5xx for each. <em>"<code>POST /orders</code> → <code>201</code> + <code>Location</code>; <code>GET /orders/&#123;id&#125;</code> → <code>200</code> or <code>404</code>."</em> <a href="../lessons/http-methods-and-status-codes.html">L2</a></li>
    <li><strong>Make writes safe: idempotency.</strong> <code>Idempotency-Key</code> on unsafe <code>POST</code>s so retries don't double-charge. <em>"Client sends a key; I dedupe server-side."</em> <a href="../lessons/idempotency-in-practice.html">L3</a></li>
    <li><strong>Collections: pagination, filtering, sorting.</strong> Cursor-based at scale, not offset. <em>"Cursor pagination so deep pages stay cheap and stable under writes."</em> <a href="../lessons/pagination-at-scale.html">L6</a></li>
    <li><strong>Caching &amp; concurrency.</strong> <code>ETag</code> + <code>Cache-Control</code>; <code>If-Match</code> for optimistic concurrency → <code>412</code> on stale write. <em>"Conditional requests prevent lost updates."</em> <a href="../lessons/caching-and-conditional-requests.html">L7</a></li>
    <li><strong>Versioning &amp; evolution.</strong> Additive first; version only on breaking change; deprecate with a <code>Sunset</code> header. <em>"I'd evolve additively before bumping the version."</em> <a href="../lessons/api-versioning-strategies.html">L5</a></li>
    <li><strong>AuthN/AuthZ + object-level.</strong> OAuth2/OIDC, JWTs, and per-object checks to prevent BOLA. <em>"I verify the caller owns <em>this</em> object, not just that they're authenticated."</em> <a href="../lessons/authentication-and-authorization.html">L8</a></li>
    <li><strong>Errors, rate limiting, observability.</strong> RFC&nbsp;9457 problem details, <code>429</code> + <code>Retry-After</code>, RED metrics (rate/errors/duration). <em>"Structured errors, and I watch p99 on each route."</em> <a href="../lessons/error-design-rate-limiting-observability.html">L9</a></li>
    <li><strong>Async / webhooks where needed.</strong> Long jobs → <code>202</code> + a status resource to poll; signed, at-least-once webhooks. <em>"It's at-least-once, so the consumer must be idempotent."</em> <a href="../lessons/async-and-long-running-operations.html">L10</a></li>
    <li><strong>Trade-offs &amp; "what changes at 100×".</strong> Name the trade-off, pick a side, state when you'd decide differently. <em>"That's a deliberate trade-off because…; at 100× I'd revisit X."</em></li>
  </ol>

  <h2>Phrases that signal senior</h2>
  <dl>
    <dd>"That's a deliberate trade-off because…" <span class="why">→ you chose, you didn't default.</span></dd>
    <dd>"It's at-least-once, so the consumer must be idempotent." <span class="why">→ you know delivery guarantees.</span></dd>
    <dd>"I'd avoid a version bump and evolve additively." <span class="why">→ you protect clients first.</span></dd>
    <dd>"I'd verify the caller owns that object, not just that they're authenticated." <span class="why">→ BOLA-aware.</span></dd>
    <dd>"I care about p99, not the mean." <span class="why">→ you measure tail latency.</span></dd>
    <dd>"I want exactly-once <em>effect</em>, not exactly-once delivery." <span class="why">→ you know the difference can't be had cheaply.</span></dd>
  </dl>
