Review guides, self-test & what to read next

Quick self-test (answers marked)

Cover the right column with your thumb, pick, then check.

1 · Who executes a tool call?

The LLM runs it internally

The runtime runs it; the model only emitted the request

The MCP protocol runs it

2 · "The model remembers our last message" is wrong because…

the runtime re-sends the history; the model is stateless

memory lives in the model weights

the context window caches between calls

3 · The difference between a workflow and an agent is…

an agent uses a bigger model

a workflow can't call tools

who writes the control flow — the developer, or the model

4 · MCP exists primarily to…

turn N×M bespoke integrations into N+M via one standard

make the model run tools faster

give one vendor exclusive tool access

5 · The agent loop is best compared to…

a database transaction

a Kubernetes reconciliation loop driving toward desired state

a message queue

6 · Single vs multi-agent maps cleanly onto…

sync vs async calls

a monolith vs a microservices architecture

a read replica vs a primary

Reconstruct the stack from memory

The real test: write the ten levels in order, one line each, with the book closed. Then check against the spine:

1 LLM stateless text→text · 2 Tool: model requests, runtime runs · 3 Agent = LLM+tools+loop+goal · 4 Loop: observe→think→act→repeat · 5 Runtime: app server + guardrails · 6 Memory: runtime reloads (RAM vs disk) · 7 Workflow vs Agent: who writes the if/else · 8 MCP: USB-C; server/client/registry; N×M→N+M · 9 Single vs Multi: monolith vs microservices · 10 Archetypes: same engine + different tools + job.

5-minute review (weekly)

Don't re-read — recall. Say the one sentence; redraw the request path and the loop from memory; recite the ten rules; translate three buzzwords blind. Anything that stalls is your cue for the deep dive.

30-minute deep dive (when a step stalled)

Blank-page reconstruct all ten levels (~8 min). Then re-derive the chain of necessity — why each layer forces the next: stateless LLM → tools → loop → runtime → memory → workflow/agent → MCP → single/multi → archetypes (~10 min). Then apply it: take one real product and place it (workflow/agent? single/multi? which archetype? which tools?) (~7 min). Re-read the myth-vs-truth lines for the two levels you missed (~5 min).
